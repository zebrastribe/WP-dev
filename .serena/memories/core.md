# wp-dev — graph root

WordPress local Docker CLI + SSH pull/push sync (staging/production). Not a hosting platform.

## Critical invariants

- **Never** overwrite DB without backup (`~/.wp-dev/backups/<project>/<env>/`).
- `pull`/`push` always run WP-CLI search-replace for URL switching.
- rsync **excludes** `wp-config.php`; table prefix via `WORDPRESS_TABLE_PREFIX` in `docker/.env`.
- SSH is **public-key only** (node-ssh); no password in config.
- Compose project id from `project` (or `local.composeProjectName`) — isolates containers/volumes per clone.
- `wordpress/` ownership: host uid for rsync; `www-data` on runtime paths (`fix-permissions` / `fix-runtime-permissions`).
